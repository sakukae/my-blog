## AI Coding 常用 prompt

### 快速了解项目结构 

```Plain Text
请阅读项目的 README.md、package.json 和主要目录，帮我了解这个项目的架构和技术栈，但暂时不要编写任何代码。
```

```Plain Text
请先阅读当前项目，不要修改任何代码。
帮我总结：
1. 项目是做什么的
2. 技术栈是什么
3. 目录结构如何组织
4. 启动、构建、测试命令是什么
5. 新增一个页面应该从哪里开始
```

### 修复 bug

```Plain Text
我遇到了一个 bug: XXX。
请先阅读相关代码，不要修改。
输出：
1. 可能原因
2. 涉及文件
3. 你需要进一步确认的信息
4. 修复计划
```

### 确认计划

```Plain Text
请按刚才确认的计划修改代码。要求：
修改范围只限于 XXX
不要引入新依赖
不要重构无关代码
完成后运行相关测试
最后总结修改内容和风险
```

### review 代码

```Plain Text
请 review 当前 git diff。
重点关注：
1. 是否破坏现有功能
2. 是否有边界情况遗漏
3. 是否有错误处理缺失
4. 是否有类型问题
5. 是否有安全风险
6. 是否需要补测试
只输出问题，不要修改代码。
```

### 任务交付

```Plain Text
请总结这个任务。
包括：
1. 目标是什么
2. 最终改了什么
3. 已验证什么
4. 还有什么风险
5. 后续继续做应该从哪里开始
```
